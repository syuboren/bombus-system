Part 1

為什麼需要結構化的職能管理？

能力定義模糊

各部門對「溝通能力」「專業技能」的定義不一，面試評核與晉升標準缺乏客觀依據。

職等薪資不透明

員工不清楚晉升路徑與條件，薪資級距缺乏系統化管理，跨子公司標準不一致。

JD 管理分散

職務說明書散落於 Word/Email，版本混亂且無法與職能框架連動，難以作為招募與評核依據。

系統願景

透過三大模組的結構化串聯，建立從「能力定義」→「階層架構」→「職務規格」的完整職能管理體系，作為招募、評核、晉升、培訓的共同基準。

蓋房子的比喻

職能模型 = 建材清單（需要哪些能力）
職等職級 = 樓層設計圖（組織有幾層）
職務說明書 = 房間規格書（每個職位的完整說明）

職能模型基準

四大職能類別

職能模型是整個職能管理體系的基石，定義組織需要的每一種「能力」。

核心
Core Competency
全體員工

組織的基本 DNA，每個人都須具備。如：溝通表達、自我管理。

L1 - L6 累積制
管理
Management
管理者

帶領團隊所需的領導能力。如：團隊領導、決策能力。

L1 - L6 累積制
專業
Professional
特定部門

該領域的專業技術能力。如：系統設計、財務分析。

L1 - L6 累積制
KSA
Knowledge / Skill / Attitude
依職務而定

知識(K)、技能(S)、態度(A) 三維度分類，無等級制度。

無等級

KSA 三維度

K — 知識

該職務需要「知道」的事。如：勞動法規、產品知識。

S — 技能

該職務需要「做到」的事。如：Excel 進階、專案管理。

A — 態度

該職務需要「展現」的特質。如：主動積極、團隊合作。

部門職位對照

每個部門在各職等、各軌道下定義具體職位名稱。如：人資部 / 職等 3 / 管理職 = 人資副理。

子公司覆蓋

職等結構 (1-7) 全集團統一，但薪資級別、頭銜、部門職位可由子公司自行定義覆蓋。

審核流程

職等職級管理的所有新增、修改、刪除操作都不會直接生效，而是進入審核流程。

變更審核流程

1
使用者提交變更 — 新增/修改/刪除職等相關資料
2
建立待審核記錄 — 狀態設為 pending，顯示於「待審核」頁籤
3
審核者操作 — 核准(approved) 或 駁回(rejected)
4
核准 → 套用變更：正式更新到資料庫
駁回 → 通知修改：記錄駁回原因

狀態流轉

pending → approved 或 rejected

審核記錄包含

  • 變更類型（新增 / 修改 / 刪除）
  • 舊資料快照 vs 新資料快照
  • 提交者與提交時間
  • 核准者與核准時間
  • 駁回原因（若被駁回）

晉升條件設定

每個「從 A 職等到 B 職等」的晉升路徑可定義具體條件，作為員工發展的明確目標。

晉升條件欄位

欄位說明範例
起始職等目前在幾等3
目標職等要升到幾等4
適用軌道管理 or 專業management
績效門檻最近評核等級A 或 B
必備技能技能清單專案管理、預算編制
必修課程訓練課程管理學基礎
KPI 重點績效指標團隊生產力 +10%
其他條件附加要求帶領 5 人以上滿一年
晉升流程流程說明自薦→推薦→審核→核准

範例：職等 2 → 職等 3（管理職）

績效 最近年度評核 ≥ B 等
技能 溝通表達 L3、團隊管理 L2
課程 基礎管理學、簡報技巧
條件 在現職等服務滿 2 年

與其他模組的連動：晉升條件中的「必備技能」連結職能模型基準，「必修課程」可連結教育訓練 (L3) 模組，形成從「缺什麼能力」→「上什麼課」→「達成晉升」的完整路徑。

審核與版本管理

JD 狀態流轉

草稿 → 審核中 → 已發佈 → 已封存
狀態可執行操作
draft編輯、送出審核、刪除
pending_review核准、退回（附理由）
rejected修改、重新送出
published封存、建立新版本
archived僅供查閱

版本管理機制

1
已發佈的 JD 需修改 → 點選「建立新版本」
2
系統自動建立新的草稿版本
3
修改後重新送出審核
4
新版本核准後取代舊版本

所有歷史版本保留完整快照，可隨時回顧比較差異。

退回機制：審核者退回時必須附上理由說明，JD 回到「已退回」狀態，撰寫者修改後可重新送出進入下一輪審核。

Roadmap

尚待宇創提供 AI 功能

以下為目前系統已規劃 UI 介面但尚未完成後端 AI 服務的功能項目。

職等職級管理
三大職涯發展路徑

分析員工的三個發展方向：垂直晉升（現職等→下一等）、橫向轉調（同職等不同角色）、跨部門發展（跨到其他部門的可能性），含時間估計與風險等級。

AI 職涯規劃助手

連接真實員工資料（職等、年資、評核、已完成課程），分析職能缺口對照晉升條件，產生個人化推薦與具體訓練計畫。目前前端 UI 完整，使用模擬資料。

職務說明書
AI 職務撰寫助手

使用者輸入職位描述，AI 自動解析並產生 11 大區塊內容、推薦職能需求（含等級與權重）、建議職等與薪資範圍。前端有進度動畫，無實際生成邏輯。

職能模型基準
AI 職能生成

根據產業類型與職位角色，AI 建議適合的職能清單。根據職能名稱自動產生 L1-L6 行為指標描述。根據 JD 內容反向推薦需要新增的職能。

缺口分析
職能落差分析

目前雷達圖與差距計算已實作，但數據依賴手動評核。後續將整合工作成果數據自動推算能力等級，並提供 AI 輔助評核建議與 360 度回饋。

Summary

系統導入的三大核心效益

能力標準化

四大職能類別 + L1-L6 六級制度，為組織建立統一的能力語言，消除定義模糊造成的溝通落差。

發展透明化

雙軌制晉升路徑 + 明確條件設定 + 審核機制，讓員工清楚看見成長方向，提升組織留才能力。

管理結構化

11 區塊 JD + 版本控管 + 審核流程，將職務規格從散落的文件升級為可追溯、可連動的結構化資產。
