1. 什麼是權限管理？為什麼需要它？

想像公司是一棟大樓：

比喻系統對應作用
大門鑰匙 登入帳號密碼 讓你進入系統
樓層門禁卡 角色權限 決定你能使用哪些功能
門禁等級 操作等級 + 資料範圍 決定你能做什麼、看多大範圍
💡
一句話總結：權限管理 = 「誰」可以對「哪些功能」做「什麼程度的操作」，看到「多大範圍的資料」。

2. 核心觀念：三層架構一次搞懂

Bombus 的權限系統由三層組成，就像疊積木一樣：

第一層 角色 (Role) 組織裡的「職務模板」 ▼ 例如：HR 經理、部門主管、一般員工 第二層 功能權限 (Feature Perm) 每個角色可以操作哪些功能、到什麼程度 ▼ 例如：員工檔案「可編輯」，招募「僅查看」 第三層 使用者指派 (Assignment) 把角色分配給真實的人 例如：王小明 → HR 經理

1
建立角色
建立職務模板
➡
2
設定功能權限
配置門禁卡
➡
3
指派給使用者
發卡給員工

3. 角色管理 — 建立組織的「職務模板」

什麼是角色？

角色就像公司裡的「職位名牌」，代表一組權限的集合。

角色名稱類比用途
超級管理員大樓管理員所有功能全開，負責系統維護
HR 經理HR 部門主管管理招募、員工檔案、人才庫
部門主管各部門經理查看部門員工資料、管理會議
一般員工普通員工查看自己的資料、參加會議

如何新增角色？

🎬 場景

公司新設了「訓練專員」職位，需要建立對應的系統角色。

  1. 1進入「系統設定 → 角色權限管理」頁面
  2. 2點擊右上角「新增角色」按鈕
  3. 3填寫角色名稱：訓練專員
  4. 4填寫說明（選填）：負責教育訓練課程規劃與執行
  5. 5點擊「建立角色」
ℹ️
角色建立後，還需要進一步設定「功能權限」（第 4 節），角色才能真正發揮作用。

角色卡片上的資訊

每張角色卡片會顯示：

  • 角色名稱 — 例如「HR 經理」
  • 系統角色 標籤 — 系統預設角色會標示
  • 已指派 N 位使用者 — 點擊可查看使用該角色的人員名單
  • 操作按鈕 — 檢視功能權限、編輯功能權限、編輯角色、刪除
⚠️
注意：系統角色無法刪除，但可以編輯名稱。如果有使用者正在使用該角色，刪除後該使用者將失去對應的權限。

4. 功能權限設定 — 決定每個角色「能做什麼」

為每個角色設定功能權限時，需要決定兩件事：

維度一：操作等級（能做什麼？）

操作等級說明類比
無權限 完全看不到這個功能 這層樓你進不去
僅查看 只能看，不能新增、修改、刪除 可以透過玻璃看，但門鎖著
可編輯 可以看、新增、修改、刪除 有鑰匙可以自由進出

維度二：資料範圍（能看/改多大範圍？）

資料範圍說明類比
個人 只能看到/編輯自己的資料 只能進自己的辦公室
部門 能看到/編輯自己部門所有人的資料 能進整個部門區域
全公司 能看到/編輯公司所有人的資料 能進大樓所有房間

重要規則：查看範圍 ≥ 編輯範圍

系統會自動確保「你能看到的範圍」不小於「你能編輯的範圍」。

設定是否合理？說明
編輯：部門、查看：全公司 ✅ 合理 能看全公司，但只能改自己部門
編輯：個人、查看：部門 ✅ 合理 能看部門資料，但只能改自己的
編輯：全公司、查看：部門 ❌ 不允許 能改全公司但只看得到部門？矛盾！

如何設定角色的功能權限？

🎬 場景

剛建立的「訓練專員」角色，需要設定他能使用哪些功能。

  1. 1在角色卡片上點擊「編輯功能權限」
  2. 2系統會展開一個權限矩陣表，包含三大模組的所有功能（如下方預覽）
  3. 3逐一調整每個功能的操作等級與範圍
  4. 4點擊「儲存功能權限」

權限矩陣預覽

功能名稱操作等級編輯範圍查看範圍
▼ L1 員工管理
招募職缺管理 僅查看 — 全公司
員工檔案管理 可編輯 部門 全公司
人才庫管理 無權限 — —
▼ L2 職能管理
職等職級矩陣 僅查看 — 全公司
💡
操作技巧：
  • 選擇「無權限」→ 編輯和查看範圍自動隱藏
  • 選擇「僅查看」→ 只需設定查看範圍
  • 選擇「可編輯」→ 需設定兩項範圍，系統會自動確保查看 ≥ 編輯

5. 使用者管理 — 把角色指派給真人

建立好角色並設定完權限後，最後一步就是把角色「發」給真正的使用者。

指派時的「作用範圍」

作用範圍說明使用時機
全域角色權限在整個組織內生效大多數情況使用此項
子公司角色權限僅在指定子公司內生效集團企業中限定管理範圍
部門角色權限僅在指定部門內生效指定某人為特定部門主管

如何為使用者指派角色？

🎬 場景

新進員工李小華入職，需要給她「HR 經理」的權限，限定在「人資部」。

  1. 1進入「系統設定 → 使用者管理」頁面
  2. 2找到「李小華」，點擊「角色管理」按鈕
  3. 3在「指派新角色」區塊，角色選擇「HR 經理」— 下方會自動顯示功能權限預覽
  4. 4作用範圍選擇「部門」，組織單位選擇「人資部」
  5. 5點擊「指派」
👁️
功能權限預覽：在選擇角色的同時，系統會立即顯示該角色的功能權限摘要，讓你在指派前確認「這個角色真的適合給這位員工嗎？」

其他操作

操作說明
移除角色點擊角色旁邊的 ✕ 按鈕即可移除
查看角色權限點擊「查看權限」展開完整的功能權限表格
有效權限（合併後）當使用者有多個角色時，底部按鈕可查看合併後的最終權限

6. 多角色合併 — 一個人有多個角色怎麼辦？

合併原則：取最大值

Bombus 採用「寬鬆合併」策略 — 當一個使用者有多個角色時，每項功能取最高的操作等級和最廣的資料範圍。

💡
類比：你同時拿著兩張門禁卡，系統會把兩張卡能進的房間都打開給你。

8. 常見情境操作指南

🎬 情境 A：新員工入職

小美剛加入行銷部，需要基本的系統使用權限。

  1. 1使用者管理 → 確認小美的帳號已建立
  2. 2點擊小美的「角色管理」
  3. 3指派角色 → 選擇「一般員工」，作用範圍「全域」
  4. 4點擊「指派」→ 完成！

結果：小美可以查看自己的員工檔案、參加會議、查看職能資料。

🎬 情境 B：員工升任部門主管

小明原本是一般員工，現在升任為行銷部主管。

  1. 1使用者管理 → 找到小明，點擊「角色管理」
  2. 2保留原有的「一般員工」角色（不需移除）
  3. 3新增指派 → 選擇「部門主管」，作用範圍「部門」→「行銷部」
  4. 4點擊「指派」

結果：小明的權限自動合併 — 除了原本的個人功能外，現在還能查看行銷部所有員工資料、管理部門會議。

🎬 情境 C：建立全新的自訂角色

公司需要一個「招募專員」角色，只負責招募相關功能。

  1. 1角色管理 → 點擊「新增角色」，名稱填「招募專員」
  2. 2建立後，點擊「編輯功能權限」
  3. 3設定：招募 / AI 面試 / 人才庫 → 可編輯（全公司）；員工檔案 → 僅查看；其餘 → 無權限
  4. 4儲存功能權限
  5. 5到使用者管理，將此角色指派給相關人員
🎬 情境 D：確認使用者的實際權限

主管想確認「李小華」目前到底有哪些權限。

  1. 1使用者管理 → 找到李小華，點擊「角色管理」
  2. 2查看「目前角色」區塊，列出所有已指派角色
  3. 3點擊各角色的「查看權限」展開細節
  4. 4點擊底部「有效權限（合併後）」查看最終結果
🎬 情境 E：暫時撤銷權限（員工請長假）

方式做法效果
停用帳號（推薦） 使用者管理 → 編輯 → 狀態改為「停用」 完全無法登入，回來時改回「啟用」即恢復
移除角色 角色管理 → 移除所有角色 可登入但無功能權限，需手動重新指派
🎬 情境 F：查看某角色被哪些人使用

  1. 1角色管理 → 找到目標角色卡片
  2. 2卡片上顯示「已指派 N 位使用者」
  3. 3點擊連結，彈出使用者清單（姓名、Email、作用範圍）

9. 常見問題 FAQ

Q1：修改角色的功能權限後，已指派的使用者會立即生效嗎？
是的。權限即時生效，所有擁有該角色的使用者在下次操作時就會套用新權限，無需登出再登入。
Q2：一個使用者可以有幾個角色？
沒有上限。系統會自動合併所有角色的權限，取最高等級和最廣範圍。
Q3：刪除角色後，已指派的使用者會怎樣？
使用者會立即失去該角色提供的所有權限。如果只有這一個角色，他將失去所有功能存取權（但仍可登入系統）。
Q4：「作用範圍」和「資料範圍」有什麼不同？
概念設定位置說明
資料範圍（個人/部門/全公司） 角色的功能權限設定 決定在功能內能看到/編輯多大範圍的資料
作用範圍（全域/子公司/部門） 使用者的角色指派 決定這個角色在哪個組織範圍內生效
Q5：系統角色和自訂角色有什麼差別？
項目系統角色自訂角色
來源系統初始化自動建立管理員手動建立
能否刪除❌ 不能✅ 可以
能否修改名稱✅ 可以✅ 可以
能否修改權限✅ 可以✅ 可以
標示有 系統角色 標籤無標籤
Q6：如果所有管理員都被移除角色，系統會怎樣？
平台管理員（Platform Admin）始終擁有最高權限，不受角色設定影響。如果發生權限設定錯誤，平台管理員可以登入並修復。
Q7：可以讓同一個角色在不同部門有不同的權限嗎？
不行。角色的功能權限是全域統一的。但你可以：
  • 建立多個角色（如「行銷部主管」、「研發部主管」），各自設定不同權限
  • 或使用同一個「部門主管」角色，但在指派時指定不同的部門
