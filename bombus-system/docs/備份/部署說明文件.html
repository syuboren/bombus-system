三、後端部署

3.1 安裝依賴

bash
# 進入後端目錄 cd bombus-system/server # 安裝生產依賴 npm install --production # 或安裝所有依賴（含開發工具） npm install

3.2 環境變數配置

在 server/ 目錄下建立 .env 檔案：

.env
# ===== 伺服器配置 ===== PORT=3001 NODE_ENV=production FRONTEND_URL=https://your-domain.com # ===== 104 API 整合（Sandbox 環境）===== API_104_BASE_URL=https://apis.104api-dev.com.tw # Job API 認證 API_104_CLIENT_ID=your_client_id API_104_CLIENT_SECRET=your_client_secret # Resume API 認證 API_104_RESUME_CLIENT_ID=your_resume_client_id API_104_RESUME_CLIENT_SECRET=your_resume_client_secret

環境變數說明

變數名稱 必填 預設值 說明
PORT 選填 3001 API 伺服器監聽 Port
NODE_ENV 選填 development 環境模式（production/development）
FRONTEND_URL 必填 - 前端 URL，用於 CORS 和連結生成
API_104_* 選填 - 104 人力銀行 API 認證（如需整合）

3.3 初始化資料庫

bash
# 初始化資料庫（首次部署必須執行） npm run init-db # 資料庫檔案位置 # server/src/db/onboarding.db

注意：資料庫檔案 onboarding.db 需要定期備份。建議設定自動備份排程。

3.4 啟動伺服器

開發環境

bash
# 使用 nodemon 自動重載 npm run dev

生產環境（直接啟動）

bash
# 直接啟動 npm start # 或 node src/index.js

生產環境（使用 PM2）

bash
# 安裝 PM2 npm install -g pm2 # 啟動應用 pm2 start src/index.js --name bombus-api # 設定開機自動啟動 pm2 startup pm2 save # 查看狀態 pm2 status # 查看日誌 pm2 logs bombus-api

PM2 配置檔案（建議）

在 server/ 目錄下建立 ecosystem.config.js：

ecosystem.config.js
module.exports = { apps: [{ name: 'bombus-api', script: 'src/index.js', instances: 1, exec_mode: 'fork', env: { NODE_ENV: 'production', PORT: 3001 }, error_file: './logs/err.log', out_file: './logs/out.log', log_date_format: 'YYYY-MM-DD HH:mm:ss Z', max_memory_restart: '500M' }] };
bash
# 使用配置檔案啟動 pm2 start ecosystem.config.js

四、前端部署

4.1 安裝依賴

bash
# 進入前端目錄 cd bombus-system # 安裝依賴 npm install

4.2 開發環境

bash
# 啟動開發伺服器（含 API 代理） npm start # 或指定 proxy 配置 ng serve --proxy-config proxy.conf.json # 開發伺服器位址 # http://localhost:4200

開發代理配置：開發環境透過 proxy.conf.json 將 /api 和 /uploads 請求代理到後端 http://localhost:3001。

4.3 建置生產版本

bash
# 標準建置 npm run build # 輸出目錄 # dist/bombus-system/browser/

建置配置說明

項目 設定值 說明
輸出目錄 dist/bombus-system/browser/ 建置後的靜態檔案
輸出雜湊 all 所有檔案加入雜湊以利快取
初始預算 2 MB 超過會顯示警告

4.4 部署靜態檔案

將 dist/bombus-system/browser/ 目錄下的所有檔案複製到 Web 伺服器的文件根目錄。

bash
# 複製到 Nginx 預設目錄 sudo cp -r dist/bombus-system/browser/* /var/www/html/ # 或複製到自訂目錄 sudo cp -r dist/bombus-system/browser/* /var/www/bombus/

五、Nginx 反向代理配置

5.1 安裝 Nginx

bash
# Ubuntu/Debian sudo apt update sudo apt install nginx # CentOS/RHEL sudo yum install nginx # macOS brew install nginx

5.2 Nginx 配置檔案

建立配置檔案 /etc/nginx/sites-available/bombus：

nginx.conf
server { listen 80; server_name your-domain.com; # 前端靜態檔案 root /var/www/bombus; index index.html; # Gzip 壓縮 gzip on; gzip_types text/plain text/css application/json application/javascript text/xml application/xml; gzip_min_length 1000; # 前端路由（SPA） location / { try_files $uri $uri/ /index.html; } # API 代理 location /api { proxy_pass http://localhost:3001; proxy_http_version 1.1; proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection 'upgrade'; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $scheme; proxy_cache_bypass $http_upgrade; # 大檔案上傳支援 client_max_body_size 50M; } # 上傳檔案服務 location /uploads { proxy_pass http://localhost:3001; proxy_set_header Host $host; proxy_set_header X-Real-IP $remote_addr; } # 靜態資源快取 location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ { expires 1y; add_header Cache-Control "public, immutable"; } }

5.3 啟用配置

bash
# 建立軟連結 sudo ln -s /etc/nginx/sites-available/bombus /etc/nginx/sites-enabled/ # 測試配置 sudo nginx -t # 重新載入 Nginx sudo systemctl reload nginx

5.4 HTTPS 配置（建議）

使用 Let's Encrypt 免費憑證：

bash
# 安裝 Certbot sudo apt install certbot python3-certbot-nginx # 申請憑證並自動配置 Nginx sudo certbot --nginx -d your-domain.com # 自動更新憑證 sudo certbot renew --dry-run

六、Docker 部署（建議）

6.1 後端 Dockerfile

在 server/ 目錄下建立 Dockerfile：

Dockerfile
FROM node:20-alpine WORKDIR /app # 複製依賴檔案 COPY package*.json ./ # 安裝生產依賴 RUN npm install --production # 複製原始碼 COPY . . # 建立上傳目錄 RUN mkdir -p uploads # 暴露 Port EXPOSE 3001 # 啟動指令 CMD ["npm", "start"]

6.2 前端 Dockerfile

在 bombus-system/ 根目錄下建立 Dockerfile：

Dockerfile
# 建置階段 FROM node:20-alpine AS build WORKDIR /app COPY package*.json ./ RUN npm install COPY . . RUN npm run build # 生產階段 FROM nginx:alpine # 複製建置結果 COPY --from=build /app/dist/bombus-system/browser /usr/share/nginx/html # 複製 Nginx 配置 COPY nginx.conf /etc/nginx/conf.d/default.conf EXPOSE 80 CMD ["nginx", "-g", "daemon off;"]

6.3 Docker Compose

在專案根目錄建立 docker-compose.yml：

docker-compose.yml
version: '3.8' services: # 後端 API api: build: context: ./bombus-system/server dockerfile: Dockerfile container_name: bombus-api restart: unless-stopped environment: - NODE_ENV=production - PORT=3001 - FRONTEND_URL=http://localhost volumes: - ./bombus-system/server/src/db:/app/src/db - ./bombus-system/server/uploads:/app/uploads networks: - bombus-network # 前端 + Nginx web: build: context: ./bombus-system dockerfile: Dockerfile container_name: bombus-web restart: unless-stopped ports: - "80:80" - "443:443" depends_on: - api networks: - bombus-network networks: bombus-network: driver: bridge

6.4 Docker 部署指令

bash
# 建置並啟動 docker-compose up -d --build # 查看狀態 docker-compose ps # 查看日誌 docker-compose logs -f # 停止服務 docker-compose down

七、部署檢查清單

7.1 後端部署檢查

  • Node.js 18+ 已安裝
  • npm 依賴已安裝 (npm install)
  • 環境變數已設定 (.env 檔案)
  • 資料庫已初始化 (npm run init-db)
  • 上傳目錄已建立且有寫入權限 (uploads/)
  • PM2 已安裝並設定開機自動啟動
  • API 健康檢查通過 (/api/health)
  • 資料庫備份排程已設定

7.2 前端部署檢查

  • 前端依賴已安裝
  • 生產版本已建置 (npm run build)
  • 靜態檔案已複製到 Web 伺服器
  • index.html 可正常存取
  • 靜態資源載入正常（JS、CSS、圖片）
  • SPA 路由正常運作

7.3 Nginx 配置檢查

  • Nginx 已安裝並啟動
  • 配置檔案語法正確 (nginx -t)
  • API 代理正常轉發
  • 上傳檔案路徑正確
  • HTTPS 憑證已設定（生產環境）
  • Gzip 壓縮已啟用

7.4 安全性檢查

  • 環境變數檔案權限已限制（600）
  • 資料庫檔案不對外公開
  • API Port（3001）不對外開放
  • 104 API 認證資訊已保護
  • CORS 已正確設定
  • 防火牆規則已設定

八、常見問題排解

Q1: API 回傳 CORS 錯誤

原因：前端與後端網域不同，且 CORS 未正確設定。

解決方案：

  1. 確認 FRONTEND_URL 環境變數已正確設定
  2. 確認 Nginx 代理配置正確
  3. 確認前端 API 請求使用相對路徑 /api

Q2: 前端路由重新整理後顯示 404

原因：SPA 路由需要伺服器配合，將所有請求導向 index.html。

解決方案：

  1. 確認 Nginx 配置包含 try_files $uri $uri/ /index.html;
  2. 或改用 Hash 路由策略

Q3: 檔案上傳失敗（413 錯誤）

原因：Nginx 預設限制請求大小為 1MB。

解決方案：

  1. 在 Nginx 配置中加入 client_max_body_size 50M;
  2. 重新載入 Nginx 配置

Q4: 資料庫鎖定錯誤

原因：SQLite 在高併發情況下可能發生鎖定。

解決方案：

  1. 確保只有一個 Node.js 程序存取資料庫
  2. PM2 設定 instances: 1
  3. 如需高併發，考慮改用 PostgreSQL

Q5: PM2 無法自動重啟

原因：PM2 startup 未正確設定。

解決方案：

pm2 startup # 執行輸出的指令 pm2 save

Q6: 104 API 連線失敗

原因：API 認證資訊錯誤或網路問題。

解決方案：

  1. 確認 .env 中的 104 API 認證資訊正確
  2. 確認伺服器可以連線到 104 API 端點
  3. 檢查 104 API 帳號狀態是否正常
