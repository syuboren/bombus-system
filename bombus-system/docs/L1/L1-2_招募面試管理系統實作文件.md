# 招募面試管理系統實作文件 (Recruitment Interview Management System)

## 1. Codebase Locations

*   **Frontend**:
    *   **Page Component**: `src/app/features/employee/pages/recruitment-page/recruitment-page.component.ts`
        *   負責面試流程管理、評分、媒體上傳、AI 分析觸發與決策制定。
    *   **Service**:
        *   `src/app/features/employee/services/interview.service.ts` (API 呼叫)
        *   `src/app/features/employee/services/ai-analysis.service.ts` (AI 分析邏輯)

*   **Backend**:
    *   **Route Handler**: `server/src/routes/recruitment.js`
    *   **Database Schema**: `server/src/db/index.js`

## 3. API Endpoints

### Candidates & Listing
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/recruitment/candidates` | 取得所有候選人列表（包含面試狀態） |
| `GET` | `/api/recruitment/candidates/:id` | 取得特定候選人詳情（包含面試、評分、AI數據） |

### Interview Management
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/api/recruitment/interviews` | 安排新面試（創建 `interviews` 記錄） |
| `POST` | `/api/recruitment/candidates/:id/invitations` | 發送面試邀請信（含多個時段選擇） |
| `POST` | `/api/recruitment/interviews/:id/generate-form` | 產生候選人填寫表單連結 (QR Code) |
| `GET` | `/api/recruitment/interviews/:id/form-status` | 查詢候選人表單填寫狀態 |

### Evaluation & Scoring
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/recruitment/candidates/:id/evaluation` | 取得面試評分細節 |
| `POST` | `/api/recruitment/candidates/:id/evaluation` | 提交/儲存面試評分（含 AI 分析結果） |
| `PATCH` | `/api/recruitment/candidates/:id/evaluation` | 更新面試評分 |
| `PATCH` | `/api/recruitment/interviews/:id/evaluation` | 簡易更新面試結果 (Pass/Fail) |

### Decision
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/api/recruitment/candidates/:id/decision` | 提交最終錄用決策 (Offer/Reject) |
| `GET` | `/api/recruitment/candidates/:id/offer-response-link` | 取得 Offer 回覆連結狀態 |

## 4. Implementation Notes

### AI 分析流程
1.  前端 `AIAnalysisService` 接收逐字稿 (`transcript`)、候選人技能與 JD 需求。
2.  執行三個維度的分析：
    *   **關鍵字匹配** (40%): 比對專有名詞庫。
    *   **語意分析** (30%): 分析自信度、團隊合作傾向等軟實力。
    *   **JD 適配度** (30%): 計算硬技能匹配率。
3.  分析結果打包為 JSON，隨 `POST evaluation` API 存入 `interview_evaluations.ai_analysis_result` 欄位。

### 人才庫自動標籤
當後端 `recruitment.js` 處理相關邏輯時（如 `autoTagTalent` 函數），會自動掃描候選人的經歷、技能與學歷，並對應 `talent_tags` 表進行標籤化（如「台成清交」、「前端開發」、「資深人員」），便於後續搜尋。

### 狀態流轉
*   **New** (新應徵) -> **Invited** (已邀請) -> **Interview** (面試中) -> **Scored/Pending Decision** (已評分/待決策) -> **Offered** (發出錄用) -> **Hired/Onboarded** (已報到)。
*   若在任一階段婉拒，系統可自動將其導入 **Talent Pool** 並標記婉拒原因。
