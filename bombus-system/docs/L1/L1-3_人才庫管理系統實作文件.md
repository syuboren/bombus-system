# 人才庫管理系統實作文件 (Talent Pool Management System)

## 1. Codebase Locations

*   **Frontend**:
    *   **Page Component**: `src/app/features/employee/pages/talent-pool-page/talent-pool-page.component.ts`
        *   負責人才列表展示、篩選、聯繫紀錄管理、標籤管理與職缺媒合分析。
    *   **Service**: `src/app/features/employee/services/talent-pool.service.ts`

## 2. 資料庫結構與欄位對照 (Database Schema & Fields)

### `talent_pool` (人才庫主表)
儲存人才的基本資料、狀態與來源。

| 欄位名稱 (English) | 欄位名稱 (Chinese) | 類型 | 說明 | 範例資料 (Example) |
| :--- | :--- | :--- | :--- | :--- |
| `id` | ID | TEXT | UUID, 主鍵 | `talent-001` |
| `candidate_id` | 原始候選人 ID | TEXT | 若來自招募流程，關聯其 ID | `cand-001` |
| `name` | 姓名 | TEXT | 人才姓名 | `王小明` |
| `skills` | 技能 | TEXT | JSON Array | `["Angular", "Node.js"]` |
| `source` | 來源 | TEXT | 104, website, referral, etc. | `104` |
| `status` | 狀態 | TEXT | active, contacted, hired, declined | `active` |
| `contact_priority` | 聯繫優先級 | TEXT | high, medium, low | `high` |
| `match_score` | 基礎媒合分 | INTEGER | 0-100 | `80` |
| `last_contact_date` | 最後聯繫日 | TEXT | ISO 8601 | `2024-03-01T10:00:00Z` |
| `contact_reminder_enabled` | 啟用聯繫提醒 | INTEGER | 0 或 1 | `1` |
| `decline_stage` | 婉拒階段 | TEXT | invite_declined, interview_declined 等 | `offer_declined` |

## 4. Implementation Notes

### 自動標籤 (Auto-Tagging)
系統會定期或在匯入時執行 `autoTagTalent` 邏輯，根據以下的關鍵字匹配規則自動上標籤：
*   **學歷**: 「國立大學」、「碩士學歷」、「台成清交」...
*   **技能**: 「前端開發」、「後端開發」、「AI/ML」...
*   **經歷**: 「資深人員」、「管理經驗」、「大廠經驗」...

### 職缺媒合 (Job Matching)
`analyze-jobs` API 會比對人才的技能 (`skills` + `candidate_specialities`) 與已發布職缺的描述/需求關鍵字。
*   **算法**: 計算 `匹配技能數 / 職缺關鍵字總數` 的比例，並加入微幅隨機變異模擬 AI 判斷的不確定性。
*   **結果**: 產生 `match_score` (0-100) 並存入 `talent_job_matches`，供前端排序顯示。

### 聯繫狀態追蹤
*   **Pending (待聯繫)**: 當開啟 `contact_reminder_enabled` 且超過 1 個月無聯繫紀錄時顯示。
*   **Contacted (已聯繫)**: 近 1 個月內有新增 `talent_contact_history` 紀錄。
*   系統會在 `GET /api/talent-pool` 時動態計算並更新狀態。
