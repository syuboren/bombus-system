## 3. API Endpoints

### Meeting CRUD
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/meetings` | 取得會議列表（支援 `start`, `end`, `type`, `scope` 過濾） |
| `POST` | `/api/meetings` | 建立新會議（包含出席者、議程、附件、提醒） |
| `GET` | `/api/meetings/:id` | 取得會議詳情（聚合 Attendee, Agenda, Conclusion, Attachment） |
| `PUT` | `/api/meetings/:id` | 更新會議 |
| `DELETE` | `/api/meetings/:id` | 取消/刪除會議（支援 `softDelete=true`） |

### Execution & Interaction
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/api/meetings/:id/check-in` | 出席者簽到（By `employeeId` or `name`） |
| `PATCH` | `/api/meetings/:id/complete` | 結束會議（更新狀態、填寫記錄、新增結論） |
| `POST` | `/api/meetings/:id/upload` | 上傳會議附件 |

### Conclusions (Task Tracking)
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/meetings/conclusions` | 取得跨會議的所有結論/待辦（支援 `status`, `department` 過濾） |
| `POST` | `/api/meetings/:id/conclusions` | 新增結論 |
| `PATCH` | `/api/meetings/conclusions/:id` | 更新結論進度與狀態 |

### Statistics
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/meetings/dashboard/stats` | 取得會議儀表板數據（總數、完成率、待辦執行率、逾期數） |

## 4. Workflows

### 建立會議 (Scheduling)
前端傳送完整的巢狀物件（Meeting + Attendees + Agenda + Attachments），後端會在一次 Transaction（或連續操作）中寫入所有關聯表。若附件已上傳（有 URL），則直接關聯；若為新檔案，需先呼叫 Upload API 或在建立後上傳。

### 會議進行與簽到 (Execution)
`MeetingPageComponent` 提供 QR Code 供掃描簽到，或由組織者手動勾選。後端 `check-in` API 會記錄簽到時間。

### 結案與追蹤 (Conclusion & Tracking)
會議結束後，組織者填寫 `notes` 並將 `agenda_item` 轉換為 `meeting_conclusions`。這些結論會成為全系統的待辦事項，可透過 `/api/meetings/conclusions` 跨會議追蹤進度，形成 PDCA 循環。
