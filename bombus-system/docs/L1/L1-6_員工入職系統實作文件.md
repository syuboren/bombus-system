# 員工入職系統實作文件 (Employee Onboarding System)

## 1. Codebase Locations

*   **Frontend**: `src/app/features/employee/pages/onboarding/`
    *   **Dashboard**: `onboarding-page` (HR 入職管理總覽)
    *   **Progress Tracking**: `onboarding-progress-page` (個別員工進度)
    *   **Wizard**: `onboarding-wizard-page` (候選人轉員工操作精靈)
    *   **Template Design**: `template-designer-page`, `onboarding-templates-page` (文件模板設計)
    *   **Employee Signing**: `onboarding-documents-page` (員工簽署介面)

*   **Backend Routes**:
    *   **HR Logic**: `server/src/routes/hr-onboarding.js` (候選人轉換、進度追蹤)
    *   **Template Management**: `server/src/routes/templates.js` (模板 CRUD、版本控制)
    *   **Signing Logic**: `server/src/routes/submissions.js` (簽署連結生成、提交)

*   **Database Schema**: `server/src/db/index.js` (Schema definitions)

## 2. 資料庫結構與欄位對照 (Database Schema & Fields)

### `templates` (入職文件模板)

| 欄位名稱 (English) | 欄位名稱 (Chinese) | 類型 | 說明 |
| :--- | :--- | :--- | :--- |
| `id` | ID | TEXT | UUID |
| `name` | 模板名稱 | TEXT | e.g. "勞動契約書" |
| `version` | 版本號 | INTEGER | 目前發布版本 |
| `pdf_base64` | PDF 檔案 | TEXT | Base64 encoded |
| `mapping_config` | 欄位設定 | TEXT | JSON (拖拉框座標均在此) |
| `is_active` | 啟用狀態 | INTEGER | 0 / 1 |
| `is_public` | 公開/預設 | INTEGER | 1 = 入職預設發送 |
| `is_required` | 必填 | INTEGER | 0 / 1 |
| `draft_mapping_config` | 草稿設定 | TEXT | 編輯中的暫存 |
| `has_draft` | 有草稿 | INTEGER | 0 / 1 |

### `submissions` (員工簽署記錄)

| 欄位名稱 (English) | 欄位名稱 (Chinese) | 類型 | 說明 |
| :--- | :--- | :--- | :--- |
| `id` | ID | TEXT | UUID |
| `template_id` | 模板 ID | TEXT | FK |
| `employee_id` | 員工 ID | TEXT | FK |
| `token` | 簽署 Token | TEXT | 唯一連結識別碼 |
| `status` | 狀態 | TEXT | DRAFT, SIGNED, COMPLETED |
| `approval_status` | 審核狀態 | TEXT | PENDING, APPROVED, REJECTED |
| `form_data` | 填寫內容 | TEXT | JSON (員工輸入的資料) |
| `signature_base64` | 簽名檔 | TEXT | Base64 image |
| `signed_at` | 簽署時間 | TEXT | ISO 8601 |
| `ip_address` | IP 位址 | TEXT | 簽署來源 IP |

### `employees` (入職相關欄位)

| 欄位名稱 (English) | 欄位名稱 (Chinese) | 說明 |
| :--- | :--- | :--- |
| `onboarding_status` | 入職狀態 | pending, in_progress, completed |
| `probation_end_date` | 試用期結束日 | yyyy-MM-dd |
| `probation_months` | 試用期月數 | INTEGER (預設 3) |
| `candidate_id` | 候選人 ID | 連結 Recruitment 系統 |
| `converted_at` | 轉正/入職時間 | ISO 8601 |

## 3. API Endpoints

### HR Onboarding (`/api/hr/onboarding`)
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/pending-conversions` | 取得待入職候選人 (Offer Accepted) |
| `POST` | `/convert-candidate` | 將候選人轉換為員工 (建立 Employee 記錄、觸發流程) |
| `GET` | `/in-progress` | 取得入職中員工列表與進度概況 |
| `GET` | `/progress/:employeeId` | 取得特定員工的詳細入職進度 (文件、表單) |

### Templates (`/api/onboarding/templates`)
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/` | 列表所有模板 |
| `POST` | `/` | 建立新模板 |
| `POST` | `/:id/new-version` | 發布新版本 (支援繼承舊版欄位) |
| `POST` | `/:id/draft` | 儲存草稿 |
| `POST` | `/:id/publish-draft` | 發布草稿為正式版 |

### Submissions / Signing (`/api/onboarding/sign`)
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/create` | 產生簽署連結 |
| `GET` | `/:token` | 員工取得待簽文件內容 (Schema & PDF) |
| `POST` | `/:token/submit` | 員工提交簽署 (Form Data & Signature) |

## 4. Workflow Logic

### 1. 候選人轉換流程 (Candidate Conversion)
1.  **Trigger**: HR 在「待入職列表」點擊轉換。
2.  **Action**: 呼叫 `/convert-candidate`。
3.  **Process**:
    *   建立 `employees` 記錄 (Status: 'probation', Onboarding: 'pending')。
    *   自動複製候選人資料 (Education, Skills) 至員工表。
    *   **自動發送文件**: 系統查找所有 `is_public=1` 且 `is_required=1` 的模板，自動為該員工建立 `submissions` 記錄並生成 Token。
    *   記錄 `employee_job_changes` (Type: 'hire')。

### 2. 模板設計與版本控制 (Template Versioning)
*   **Draft Mode**: 編輯時不影響線上版本，儲存於 `draft_mapping_config`。
*   **Publish**: 發布時將 Draft 寫入 `mapping_config` 並 `version + 1`，同時寫入 `template_versions` 歷史表。
*   **Legacy Support**: 已發送但未簽署的 Submission，開啟時會自動更新為最新版；已簽署的 Submission 則讀取對應的 `template_version` 歷史快照。

### 3. 員工簽署流程 (Signing)
1.  員工收到連結 (含 Token)。
2.  前端呼叫 `GET /:token` 取得 PDF 與欄位定義。
3.  員工在瀏覽器填寫、簽名。
4.  提交時 `POST /:token/submit`，系統鎖定該 Token 狀態為 `SIGNED`。
5.  HR 端看到狀態變更，可進行 `CHECK` 或 `APPROVE`。
