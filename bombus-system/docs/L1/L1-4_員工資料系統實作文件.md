# 員工資料系統實作文件 (Employee Profile System)

## 1. Codebase Locations

*   **Frontend**:
    *   **List Page**: `src/app/features/employee/pages/profile-page/profile-page.component.ts`
        *   負責員工列表展示、搜尋篩選、狀態統計與 ROI 圖表。
    *   **Detail Page**: `src/app/features/employee/pages/profile-detail-page/profile-detail-page.component.ts` (註：此元件似乎主要用於招募候選人詳情，員工詳情可能共用或位於 `profile-page` 的 Modal/Tab 中，需依實際 UI確認，目前代碼顯示 `profile-page` 透過 `selectEmployee` 載入詳情)
    *   **Service**: `src/app/features/employee/services/employee.service.ts`

*   **Backend**:
    *   **Route Handler**: `server/src/routes/employee.js`
    *   **Database Schema**: `server/src/db/index.js`

## 3. API Endpoints

### Employee Management
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/employee/list` | 取得員工列表（支援部門、狀態過濾） |
| `GET` | `/api/employee/:id` | 取得單一員工完整資料（含學歷、技能、異動、薪資） |
| `GET` | `/api/employee/stats` | 取得員工統計（總數、在職、試用期、平均年資、到期文件） |
| `GET` | `/api/employee/departments` | 取得所有部門清單 |

### Documents & Files
| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/employee/documents` | 取得員工文件列表 |
| `POST` | `/api/employee/documents` | 上傳文件 |
| `DELETE` | `/api/employee/documents/:id` | 刪除文件 |
| `GET` | `/api/employee/documents/progress` | 取得文件上傳進度 |
| `GET` | `/api/employee/expiring-documents` | 取得 30 天內到期的證照文件 |

### Onboarding Support
| Method | Endpoint | Description |
| :--- | :--- | :--- |

| `GET` | `/api/employee/progress` | 取得入職進度 |
| `GET` | `/api/employee/submissions` | 取得入職文件提交紀錄 |

### Analytics
| Method | Endpoint | Description |

## 4. Implementation Notes

### 員工詳情聚合 (Data Aggregation)
`GET /api/employee/:id` 是聚合型 API，一次回傳：
*   基本資料 (`employees`)
*   學歷 (`employee_education`)
*   技能 (`employee_skills`) - 純字串陣列
*   證照 (`employee_certifications`)
*   工作歷史 (`employee_job_changes`)
*   最新薪資 (`employee_salaries`) - 僅回傳有效且最新的那一筆 (`LIMIT 1`)

### 權限控制 (Role-Based Access)
在 `employees` 表中有 `role` 欄位：
*   `employee`: 一般員工，僅能查看自己的資料（部分 API 需驗證 `employee_id`）。
*   `manager`: 主管，可查看下屬或全公司資料（視系統實作而定）。
*   目前 API 層面尚未強制實作 JWT Middleware 攔截，主要依賴前端路由與參數傳遞。

### 檔案上傳 (File Upload)
使用 `multer` 處理檔案上傳：
*   儲存路徑：`uploads/documents/`
*   檔名規則：`doc-{timestamp}-{random}.{ext}`
*   支援類型：圖片 與 PDF
*   中文檔名處理：後端有特別處理 `latin1` 到 `utf8` 的轉換以防止亂碼。
