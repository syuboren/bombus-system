### **第一部分：基本業務邏輯 (Business Logic)**

| 評估項目 | 具體需求說明（以「人才晉升流程」為例） |
| :---- | :---- |
| **功能名稱** | 員工晉升提名、績效評核收集、委員會簽核與職等生效 |
| **目標使用者** | 申請員工（自薦/履歷補充）、直屬主管（提名/初評）、晉升委員會（決策審查）、HR（流程控管與發佈） |
| **標準操作流程** *(請條列步驟)* | 1\. HR 於系統開啟「年度晉升專案」並設定收件期限。 2\. 主管發起提名，並填寫候選人推薦表。 3\. 系統自動收集候選人近兩年考績與出勤紀錄，打包為數位審查檔。 4\. 晉升委員會進行線上評分與留言決策。 5\. 總經理最終核定。 6\. HR 派發電子晉升通知，並連動更新員工主檔的「職稱、職等、薪級」。 |
| **資料輸入欄位** *(使用者需填寫)* | 1\. 擬晉升職缺與目標職等 (下拉選單) 2\. 推薦理由與具體戰功 (長文字框，必填) 3\. 委員會評分表 (自訂評分維度，如：領導力 1-5 分) 4\. 專業證照或佐證作品集 (檔案上傳，選填) |
| **資料輸出/報表** | 1\. 晉升委員會審核總表 (匯出 Excel 供會議討論使用)。 2\. 電子晉升派令/賀信 (PDF 格式)。 3\. 員工個人職涯發展軌跡紀錄。 |
| **例外與邊界處理** *(流程失敗/退回機制)* | 1\. 若委員會「駁回」晉升，系統需發送婉拒信，且相關紀錄需保密，僅 HR 與直屬主管可見。 2\. 晉升生效日若落在「計薪週期中（破月）」，系統需自動通知「薪資模組」進行依天數比例的底薪拆分計算。 |

### **第二部分：SaaS 專屬架構設定 (SaaS Architecture)**

| 評估項目 | 請勾選或說明對應的 SaaS 邏輯 |
| :---- | :---- |
| **適用訂閱方案** *(此功能誰能用？)* | ☐ 基礎版 (Basic) ☐ 專業版 (Pro) ☑ 企業版 (Enterprise) *(註：多階層自訂簽核與績效連動屬進階應用，建議綁定高階訂閱方案)* |
| **規則自訂程度** *(各家買家能否修改)* | ☐ **全平台統一 (硬邏輯)**：所有租戶固定為「主管 ➔ 總經理」兩關。 ☑ **完全自訂 (引擎)**：各租戶 (購買廠商) 可自由在後台設定「晉升簽核路線」，包含：串聯會辦、設定特定職級才能審核、自訂委員會名單，以及自由新增「評分表單欄位」。 |
| **租戶預設資料** *(新客戶剛買時)* | 系統僅提供一份「標準晉升推薦表」公版。簽核流程預設為空，新租戶首次使用時，系統必須強制引導 HR 進入「流程設定中心」設定專屬的晉升路線。 |
| **功能用量上限** *(防止單一買家塞爆系統)* | 佐證作品集上傳限制：單一員工附件總容量不超過 20MB；支援 PDF 與常見圖片格式，不支援影音檔上傳以節省雲端空間。 |

### **第三部分：全域管理與自動化 (Super Admin & Automation)**

| 評估項目 | 具體需求說明 |
| :---- | :---- |
| **觸發通知機制** *(何時? 給誰? 管道?)* | 1\. **專案啟動時**：系統自動推播通知全公司主管「年度晉升提名開始」。 2\. **核定通過時**：發送精美晉升電子賀信給員工本人，並可設定是否同步發布於「公司內部公告欄」。 |
| **超級後台權限** *(系統營運方可見度)* | ☑ **絕對隱私 (Zero Knowledge)**：晉升涉及最高級別的績效評價與薪資變動，系統營運方（超級管理員）**絕對不可見**任何評核內容與升遷名單。 ☑ **僅看狀態**：營運方後台僅顯示統計數據，例如「A 廠商本月啟動了 1 次晉升專案，共處理 15 筆審核」（做為 SaaS 活躍度分析指標）。 |
| **客製化外觀體驗** | ☑ 員工收到的 PDF 晉升派令/賀信，支援租戶上傳「公司印章/浮水印」與「專屬 Banner 圖片」，確保符合各企業的雇主品牌 (Employer Branding) 形象。 |
