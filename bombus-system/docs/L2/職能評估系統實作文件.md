# 職能評估 (Competency Assessment) 系統實作文件

本文件記錄 `competency/assessment` 功能的技術實作細節，涵蓋月度檢核、季度面談與工作週報三大模組。

## 1. 程式碼位置 (Codebase Locations)

### 前端 (Frontend)
- **頁面組件**: `src/app/features/competency/pages/assessment-page/assessment-page.component.ts`
- **頁面模板**: `src/app/features/competency/pages/assessment-page/assessment-page.component.html`
- **資料模型**: `src/app/features/competency/models/assessment.model.ts`
- **資料服務**: `src/app/features/competency/services/assessment.service.ts`
- **相關模態框 (Modals)**:
    - `src/app/features/competency/components/monthly-check-modal`
    - `src/app/features/competency/components/quarterly-review-modal`
    - `src/app/features/competency/components/weekly-report-modal`

### 後端 (Backend)
- **API 路由**:
    - 月度檢核: `server/src/routes/competency.js` (含統計 API)
    - 季度面談: `server/src/routes/quarterly-reviews.js`
    - 工作週報: `server/src/routes/weekly-reports.js`
    - 指標模板: `server/src/routes/monthly-check-templates.js`
- **資料庫檔案**: `server/src/db/onboarding.db` (SQLite)

## 3. 系統流程與邏輯 (System Workflow & Logic)

### 3.1 月度檢核流程
1.  **建立 (Create)**: 每月由系統自動或手動建立，狀態為 `self_assessment`。可選擇複製上月模板或使用預設職位模板。
2.  **自評 (Self Assessment)**: 員工填寫 `self_score`，完成電子簽名後提交 -> 狀態轉為 `manager_review`。
3.  **主管審核 (Manager Review)**:
    *   **核准**: 主管填寫 `manager_score`，完成電子簽名後提交 -> 狀態轉為 `hr_review`。
    *   **退回**: 狀態退回至 `self_assessment`，清除員工簽名需重簽。
4.  **HR 結案 (HR Close)**:
    *   **結案**: HR 確認無誤，完成電子簽名 -> 狀態轉為 `completed`，系統計算最終 `total_score`。
    *   **退回**: 可退回至主管審核階段 (`manager_review`)。

### 3.2 季度面談流程
1.  **初始化 (Initialize)**: 每季初由 HR 批量建立，狀態為 `pending`。系統自動計算該季度的「月度考核平均分」。
2.  **員工提交 (Employee Submit)**: 員工填寫自評區塊 (KPI, OKR, 職能自評等) 與滿意度調查 -> 狀態轉為 `employee_submitted`。
3.  **主管評核 (Manager Review)**: 主管檢視員工自評，填寫主管評語與發展建議 -> 狀態轉為 `manager_reviewed`。
4.  **預約面談 (Schedule Interview)**: 設定面談日期與地點 -> 狀態轉為 `interview_scheduled`。
5.  **完成面談 (Complete Interview)**: 面談結束後，主管輸入最終評分 (`total_score`) -> 狀態轉為 `interview_completed`。
6.  **HR 結案 (Completed)**: HR 確認歸檔 -> 狀態轉為 `completed`。

### 3.3 工作週報流程
1.  **生成 (Generate)**: 每週一自動生成，狀態為 `not_started`。
2.  **填寫 (Draft)**: 員工開始編輯後狀態轉為 `draft`。需填寫各類工作項目並追蹤時數。
3.  **提交 (Submit)**: 員工完成填寫並電子簽名 -> 狀態轉為 `submitted`。
4.  **審核 (Review)**:
    *   **通過**: 主管簽名確認 -> 狀態轉為 `approved`。
    *   **退回**: 主管填寫退回原因 -> 狀態轉為 `rejected`，員工需修改後重送。
