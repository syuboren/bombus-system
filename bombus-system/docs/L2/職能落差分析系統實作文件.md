## 4. 實作備註 (Implementation Notes)

1.  **落差計算邏輯 (Gap Calculation Logic)**:
    *   `Gap = Required Level (JD) - Actual Level (Assessment)`
    *   若 `Gap > 0`: 表示能力不足 (Needs Improvement)。
    *   若 `Gap <= 0`: 表示能力達標或超越 (Meet/Exceed Expectation)。

2.  **嚴重程度分級 (Severity Levels)**:
    *   `critical`: Gap >= 2 (落差 2 個等級以上)
    *   `moderate`: 1 <= Gap < 2 (落差 1-2 個等級)
    *   `minor`: 0 < Gap < 1 (落差 1 個等級內)
    *   `none`: Gap <= 0 (無落差)

3.  **資料來源**:
    *   **Required Level**: 來自 `job_descriptions` table 的 `competency_requirements` 欄位 (JSON)。
    *   **Actual Level**: 來自 `quarterly_reviews` 或 `competency_assessments` table 的最新一期評估結果。
