# 職務說明書 (Job Description) 系統實作文件

本文件記錄 `competency/job-description` 功能的技術實作細節，包括程式碼位置、資料庫結構與欄位對照。

## 1. 程式碼位置 (Codebase Locations)

### 前端 (Frontend)
- **頁面組件**: `src/app/features/competency/pages/job-description-page/job-description-page.component.ts`
- **頁面模板**: `src/app/features/competency/pages/job-description-page/job-description-page.component.html`

| `grade` | 職等 | INTEGER | 如 `4` (對應 L4) | `4` |
| `grade_code` | 職等代碼 | TEXT | 如 `P4` | `P4` |

## 3. API 介面 (API Endpoints)

### 3.1 基本 CRUD
| HTTP 方法 | 路徑 | 說明 |
| :--- | :--- | :--- |
| `GET` | `/api/job-descriptions` | 取得列表 (支援 `status`, `department` 篩選) |
| `GET` | `/api/job-descriptions/:id` | 取得單筆詳情 |
| `POST` | `/api/job-descriptions` | 建立新職務說明書 |
| `PUT` | `/api/job-descriptions/:id` | 更新內容 |
| `DELETE` | `/api/job-descriptions/:id` | 刪除 |
| `GET` | `/api/job-descriptions/generate-code` | 產生職位代碼 (需 `department`, `grade`) |

### 3.2 狀態流程與版本控制
| HTTP 方法 | 路徑 | 說明 |
| :--- | :--- | :--- |
| `POST` | `/api/job-descriptions/:id/submit-review` | 送出審核 (Draft -> Pending Review) |
| `POST` | `/api/job-descriptions/:id/approve` | 審核通過 (Pending Review -> Published) |
| `POST` | `/api/job-descriptions/:id/reject` | 退回 (Pending Review -> Rejected) |
| `POST` | `/api/job-descriptions/:id/archive` | 封存 (Published -> Archived) |
| `POST` | `/api/job-descriptions/:id/create-new-version` | 建立新版本 (Published -> Draft, 版號+1) |

### 3.3 歷史記錄
| HTTP 方法 | 路徑 | 說明 |
| :--- | :--- | :--- |
| `GET` | `/api/job-descriptions/:id/versions` | 取得版本列表 |
| `GET` | `/api/job-descriptions/:id/versions/:versionId` | 取得特定版本快照 |
| `GET` | `/api/job-descriptions/:id/approvals` | 取得審核歷程 |

## 4. 實作細節備註 (Implementation Details)

1.  **JSON 欄位處理**:
    *   後端 `server/src/routes/job-descriptions.js` 使用 `JSON.stringify` 將陣列/物件存入 SQLite 的 `TEXT` 欄位。
    *   讀取時使用 `rowToJobDescription` 函數將 JSON 字串解析回物件/陣列。
    *   前端 Angular Model (`JobDescription`) 對應解析後的結構。

2.  **狀態流轉邏輯**:
    *   **草稿 (Draft)**: 可自由編輯。
    *   **審核中 (Pending Review)**: 鎖定不可編輯，等待主管/HR 核准。
    *   **已發佈 (Published)**: 正式生效版本，不可直接修改。若需修改需「建立新版本」。
    *   **已退回 (Rejected)**: 需修改後重新送審。
    *   **已封存 (Archived)**: 歷史資料，僅供唯讀。

3.  **職能關聯**:
    *   JD 不直接儲存職能詳細內容，而是儲存 `Requirements` (需求)，包含職能 ID (`competencyId`)、需求等級 (`requiredLevel`) 與權重 (`weight`)。
    *   前端需透過 `CompetencyService` 另行取得職能主檔 (`core`, `management`, `ksa`) 來顯示職能名稱與詳細定義。
